# 📘 멋쟁이사자처럼 14기 백엔드 스터디
## GitHub 과제 제출 규칙

안녕하세요!  
백엔드 스터디 과제는 **GitHub Pull Request(PR)** 를 통해 제출합니다.

과제 제출은 조직 저장소에 직접 파일을 올리는 방식이 아니라,  
**조직 저장소를 Fork한 뒤 자신의 저장소에서 작업하고, 조직 저장소로 PR을 보내는 방식**으로 진행합니다.

---

## ✅ 1. 과제 제출 방식

과제 제출 흐름은 아래와 같습니다.

조직 저장소 Fork  
→ 내 저장소 Clone  
→ 새 브랜치 생성  
→ 과제 코드 작성  
→ Commit  
→ Push  
→ 조직 저장소로 PR 생성

즉, 과제 제출은 단순 파일 업로드가 아니라 **PR 생성**입니다.

---

## ✅ 2. 저장소 사용 규칙

조직 저장소의 `main` 브랜치에는 직접 업로드하지 않습니다.  
모든 과제는 반드시 자신의 GitHub 계정으로 Fork한 저장소에서 작업합니다.

작업이 끝난 후에는 자신의 저장소에서 조직 저장소로 Pull Request를 생성합니다.

본인 저장소의 작업 브랜치 → 조직 저장소의 main 브랜치

예시:

wooseok/backend-study의 feature/wooseok-week1  
→ likelion/backend-study의 main

---

## ✅ 3. 폴더 구조 규칙

과제 파일은 반드시 아래 구조에 맞게 제출합니다.

weekN/팀명/이름/파일명

예시:

week1/1팀/류우석/Main.java  
week1/1팀/김민수/Main.java  
week1/2팀/박지윤/Main.java  
week2/1팀/류우석/Main.java

설명:

week1 = 1주차 과제 폴더  
1팀 = 본인이 속한 팀 폴더  
류우석 = 본인 이름 폴더  
Main.java = 제출 파일

조직 저장소에 미리 만들어진 **팀/이름 폴더 구조**를 기준으로,  
자신의 폴더 안에 과제 파일을 작성한 뒤 PR을 생성합니다.

예를 들어 1팀 류우석이 1주차 과제를 제출한다면 아래와 같이 작성합니다.

week1/1팀/류우석/Main.java

---

## ✅ 4. 브랜치명 규칙

작업 전 반드시 새 브랜치를 생성합니다.

브랜치명은 아래 형식을 사용합니다.

feature/이름-weekN

예시:

feature/wooseok-week1  
feature/minsu-week2  
feature/jiyoon-week3

브랜치 생성 예시:

git checkout -b feature/wooseok-week1

---

## ✅ 5. 커밋 메시지 규칙

커밋 메시지는 아래 형식을 권장합니다.

[주차] 이름 과제 제출

예시:

[1주차] 류우석 과제 제출  
[2주차] 김민수 과제 제출

커밋 예시:

git add .  
git commit -m "[1주차] 류우석 과제 제출"

---

## ✅ 6. PR 제목 규칙

PR 제목은 반드시 아래 형식으로 작성합니다.

[주차] 이름 과제 제출

예시:

[1주차] 류우석 과제 제출  
[2주차] 김민수 과제 제출  
[3주차] 박지윤 과제 제출

PR 내용에는 필요한 경우 아래 내용을 간단히 작성합니다.

- 과제 주차
- 작성한 파일
- 어려웠던 점 또는 질문

예시:

- 1주차 백엔드 과제 제출합니다.
- week1/1팀/류우석/Main.java 파일 작성했습니다.
- 실행 과정에서 궁금한 점은 없습니다.

---

## ✅ 7. 제출 시 주의사항

아래 규칙을 반드시 지켜주세요.

main 브랜치 직접 업로드 ❌  
조직 저장소에 바로 파일 추가 ❌  
본인 저장소에서 작업 후 PR 제출 ✔  
새 브랜치 생성 후 작업 ✔  
폴더 구조 준수 ✔  
브랜치명 규칙 준수 ✔  
PR 제목 규칙 준수 ✔

## 💡 용어 설명

### Fork

조직 저장소를 자신의 GitHub 계정으로 복사하는 것입니다.  
과제는 조직 저장소에서 직접 작업하지 않고, Fork한 자신의 저장소에서 진행합니다.

### Branch

브랜치는 작업을 진행하는 개인 작업 공간입니다.  
과제마다 새 브랜치를 만들어 작업해야 합니다.

예시:

feature/wooseok-week1

### Pull Request, PR

PR은 자신의 저장소에서 작업한 내용을 조직 저장소에 반영해달라고 요청하는 과정입니다.  
스터디 과제에서는 PR 생성이 과제 제출을 의미합니다.

### main 브랜치

최종 코드가 모이는 기본 브랜치입니다.  
직접 수정하거나 바로 업로드하면 안 됩니다.

### 팀/이름 폴더

조직 저장소에 미리 만들어진 팀별 개인 제출 공간입니다.  
자신이 속한 팀과 이름 폴더를 찾아 그 안에 과제를 작성합니다.

예시:

week1/1팀/류우석/Main.java

---

## 🔥 핵심 정리

폴더 = 과제를 제출하는 위치  
브랜치 = 개인 작업 공간  
PR = 과제를 제출하는 과정  
main 브랜치 = 최종 코드가 모이는 공간  
Fork = 조직 저장소를 내 계정으로 복사하는 것

최종 제출 흐름은 아래와 같습니다.

1. 조직 저장소를 Fork한다.
2. Fork한 내 저장소를 Clone한다.
3. 새 브랜치를 만든다.
4. weekN/팀명/이름/파일명 구조로 과제를 작성한다.
5. Commit 후 Push한다.
6. 조직 저장소로 PR을 생성한다.

---

## 👉 최종 제출 기준

과제 제출은 **파일 업로드가 아니라 PR 생성**입니다.

PR이 생성되어야 과제 제출이 완료됩니다.

감사합니다!
